# Performance-First Fantasy World Gen: Development Map

**Use this map when** the primary pipeline (density-based Larion in [DEVELOPMENT_MAP.md](DEVELOPMENT_MAP.md)) is **too costly** on 1.20.1 and you decide to open the alternative track (see [IMPLEMENTATION_PLAN.md](IMPLEMENTATION_PLAN.md) §5). This design **replaces** terrain shape computation with a custom chunk generator and a **small, fixed formula**—no density tree.

---

## Performance goals

| Goal | Meaning |
|------|--------|
| **Column-scale cost** | Work per chunk is O(chunk_columns) or O(chunk_columns × small_constant), not O(vertical_cells × tree_depth). |
| **Few noise samples** | 4–8 2D noise calls per column (continent, warp X/Z, ridge, etc.); optional 3D only for caves. No DensityFunction tree. |
| **No density pipeline** | Do not use NoiseBasedChunkGenerator or the full noise_router / density-function stack for terrain shape. |
| **Keep compatibility** | Keep vanilla biomes, surface rules, and carvers so grass, snow, caves, and datapacks still work. |
| **Fantasy look** | Preserve “warped” continental shapes and strong elevation (same idea as flat_domain_warp, implemented in generator code). |

---

## Design principles

1. **Replace the generator, not the density tree**  
   One custom `ChunkGenerator` (e.g. `FantasyChunkGenerator`) produces terrain shape. Vanilla `NoiseBasedChunkGenerator` is not used for that dimension/world type.

2. **Simple, fast terrain model**  
   Per (x, z) column (or per 2×2): a small formula using 2–4 2D noises (continent, warp X/Z, ridge) → “surface Y”. Fill stone below that Y; run vanilla surface rules and carvers on the chunk.

3. **Data structures**  
   - One height value per column (2D heightmap) so we never re-evaluate the expensive part for the same column.  
   - Chunk-local buffers (e.g. `double[]` or small arrays) reused per chunk, no allocations in hot loops.

4. **What we keep vs replace**

   | Keep (vanilla / existing) | Replace (our mod) |
   |---------------------------|--------------------|
   | Biomes, biome source      | Terrain shape (height + warp) |
   | Surface rules             | How high the “stone” column goes |
   | Carvers (caves, etc.)     | Optional: our own cave noise, or keep vanilla carvers |
   | Dimension type, world height | Same height; blocks filled by our logic |
   | (Optional) Mixins for lava level, cave bound, steep slopes | Same as current Larion if we keep same dimension |

## Phase 1: Generator shell

- [x] **Create `FantasyChunkGenerator`**  
  Class that implements `ChunkGenerator` (or Forge’s interface if different). Constructor takes: `BiomeSource`, world seed (or `RandomState`), and your **config** (height scale, warp scale, min/max Y, etc.).
- [x] **Implement `fillChunk` (or equivalent)** — `fillFromNoise` (async)  
  Method that:
  1. For the chunk’s X/Z range, compute **surface Y per column** (stub: constant or simple noise for now).
  2. Fill chunk section blocks: **stone** below surface Y, **air** at and above (within world bounds).
  3. **Do not** run biomes/surface/carvers yet—just get the fill loop correct and the chunk accepted by the engine.
- [x] **Wire the generator into the world** — world preset `larion_lite:performance_overworld`  
  So that at least one dimension or world type uses `FantasyChunkGenerator` instead of `NoiseBasedChunkGenerator` (e.g. via dimension type JSON or code that replaces the generator). Confirm the game loads and chunks are generated (even if flat or ugly).
- **Deliverable:** Chunks are generated by `FantasyChunkGenerator` and rendered; terrain shape is placeholder.

## Phase 2: Terrain model (2D heightmap)

- [x] **Define a 2D heightmap per chunk**  
  e.g. `int[]` or `double[]` of size 16×16 (one value per column). Allocate once per chunk or reuse from a chunk-scoped buffer; do not allocate per column in the hot path.
- [x] **Implement “base height” per column** (+ warp + ridge; see Phase 4)  
  For each (x, z) in the chunk:
  - Sample 1–2 2D noises (e.g. “continent” and/or “base elevation”) at a chosen scale (e.g. xz_scale 0.01–0.1).
  - Map the result to a world Y range (e.g. min_y + 64 to min_y + 256, or from config).
  - Write to heightmap[columnIndex].
- [x] **Use the heightmap in the fill pass**  
  In `fillChunk`, for each column use heightmap to decide up to which Y to fill stone (and above that, air). No redundant noise calls in the fill loop.
- **Deliverable:** Terrain has simple, noise-driven elevation; one height per column; fill is a tight loop over blocks using the heightmap.

---

## Phase 3: Run vanilla biomes, surface, and carvers

- [x] **After filling stone/air, run biome assignment** — default `createBiomes` + multi_noise preset  
  Use the chunk’s biome source to set the biome for each column (or use the engine’s existing API that consumes a filled chunk and assigns biomes). Ensure the chunk has correct biomes for the rest of the pipeline.
- [x] **Run surface rules** — **lite path**: biome-tag-based tops (grass/sand/snow/water), not full `SurfaceSystem`+`NoiseChunk` (avoids density-router cost).
- [x] **Run carvers** — **Phase 5 Option B**: 3D cave noise in `fillFromNoise` (not vanilla carvers). Config `caves_enabled`.
- **Deliverable:** Playable surface + optional cheap caves.

---

## Phase 4: Warp and “fantasy” shape

- [x] **Add warp sampling in code**  
  For each column (x, z):
  - Sample 2 more 2D noises: **warp_x**, **warp_z** (e.g. scale 0.02–0.1).
  - Compute **warped** (x′, z′) = (x + warp_x, z + warp_z) (scale warp by a configurable factor, e.g. 200–800).
  - Sample your **base height** (or continent) noise at (x′, z′) instead of (x, z).
  - Use that as the column’s surface Y (or blend with unwarped for subtler effect).
  - This replicates the **idea** of flat_domain_warp (sample at offset coordinates) in one place, without a density tree.
- [x] **Optional: ridge / valley**  
  Add one more 2D noise (e.g. “ridge”) and combine with base + warp (e.g. add ridge to height, or use as mask). Keep the total number of 2D samples per column small (e.g. ≤ 6).
- [x] **Tune scales and bounds** — `FantasyTerrainConfig` + optional JSON `terrain` on generator
- **Deliverable:** Terrain has warped, continental-style shape and optional ridges; still O(columns × few) noise calls; no density functions.

## Phase 5: Caves (optional)

- [ ] **Option A: Keep vanilla carvers**  
  Ensure carvers run on chunks from `FantasyChunkGenerator` (already done in Phase 3 if you added them). Use existing mixins (lava level, cave bound) if the same dimension is used.
- [ ] **Option B: Simple 3D cave noise**  
  In the fill pass (or a second pass), for each block (x, y, z) sample one 3D noise (e.g. “cave”); if value > threshold, set to air. Limit 3D sampling to a vertical band (e.g. min_y to surface Y) to keep cost bounded. Optionally combine with vanilla carvers.
- **Deliverable:** Caves present; either vanilla carvers or a simple 3D noise; performance still acceptable.

## Class / component sketch (for implementation)

- **`FantasyChunkGenerator`**  
  Implements `ChunkGenerator`. Holds: `BiomeSource`, config (or `FantasyTerrainConfig`), `RandomState` or seed. Methods: `fillChunk` (or override the correct Forge/Minecraft method that fills the chunk), `getBaseHeightAt(x,z)`, and helpers for warp sampling.
- **`FantasyTerrainConfig`**  
  Data class or config spec: `heightScale`, `warpScale`, `minY`, `maxY`, `continentScale`, `ridgeScale`, optional `caveThreshold`. Loaded from file or preset.
- **`FantasyHeightmap`**  
  Either a 16×16 array inside the generator (reused per chunk) or a small helper that fills it from 2D noises (continent, warp, ridge). No 3D density tree.
- **Integration point**  
  Where the game chooses the chunk generator for a dimension: e.g. dimension type JSON with a custom generator type, or Forge event/code that replaces the generator when “fantasy mode” is on.

# Larion World Generation (1.20.1) – Architecture & Process Report

## 1. Project structure and build

### 1.1 Multi-loader layout

Larion is a **multi-loader** mod: one codebase, three loaders.

| Module    | Role |
|----------|------|
| **common** | Shared code and resources. No Minecraft dependency at compile time; exposes `commonJava` (source dirs) and `commonResources` (resource dirs) as consumable artifacts. |
| **forge** | Forge 1.20.1. Depends on `common`, compiles common Java + Forge, packages common resources + mixins into the JAR. |
| **fabric** | Fabric 1.20.1. Same idea with Loom. |
| **neoforge** | NeoForge (legacy Forge). Same idea. |
| **buildSrc** | Gradle plugins: `multiloader-common` (defines common artifacts), `multiloader-loader` (adds common source + resources to each loader’s `main` source set and JAR). |

Loader modules **do not** depend on each other; they only depend on `common`. The Forge build compiles `common`’s Java as part of `forge:sourceSets.main` and adds common resources to the JAR.

### 1.2 What lives in common

- **Package `com.badgerson.larion`**: Constants, CommonClass, MaterialRuleContextExtensions, SomewhatSteepMaterialCondition, SomewhatSteepSlopePredicate.
- **Package `com.badgerson.larion.density_function_types`**: All custom density function types (Division, FlatDomainWarp, Signum, Sine, Sqrt, XCoord, ZCoord).
- **Package `com.badgerson.larion.mixin`**: All mixins (NoiseChunkGeneratorMixin, SteepSlopePredicateMixin, MaterialRuleContextMixin, CaveWorldCarverMixin, SurfaceRulesContextAccessor).
- **Package `com.badgerson.larion.platform`**: Services (service loader); **platform.services**: IPlatformHelper (implemented per loader in forge/fabric/neoforge).

Resources in common:

- `larion.mixins.json` (mixin config).
- `META-INF/accesstransformer-common.cfg` (used by Forge; loader may add its own AT).
- `data/minecraft/` and `data/larion/` (dimension types, noise settings, density functions, carvers, noise definitions, etc.).

## 2. Mod entry and registration (Forge)

### 2.1 Entry point

- **Class**: `com.badgerson.larion.LarionMod` (in **forge** module).
- **Annotation**: `@Mod(Constants.MOD_ID)` with `MOD_ID = "larion"`.
- **Constructor** runs at mod init and:
  1. Registers **density function types** with Forge’s `DeferredRegister` for `Registries.DENSITY_FUNCTION_TYPE`.
  2. Registers **material conditions** for `Registries.MATERIAL_CONDITION`.
  3. Calls `CommonClass.init()` (currently no-op beyond comments).

### 2.2 Registered density function types

Each registration maps a **JSON type name** (used in datapacks) to a **Codec** that deserializes the function:

| JSON type (e.g. in "type": "larion:...") | Class    | Purpose (brief) |
|------------------------------------------|----------|------------------|
| `larion:div`                             | Division | argument1 / argument2 (0 → 0). |
| `larion:sqrt`                            | Sqrt     | sqrt(argument), ≤0 → 0. |
| `larion:signum`                          | Signum   | signum(argument). |
| `larion:sine`                            | Sine     | sin(argument). |
| `larion:x`                               | XCoord   | blockX clamped to world bounds. |
| `larion:z`                               | ZCoord   | blockZ clamped to world bounds. |
| `larion:flat_domain_warp`                | FlatDomainWarp | Evaluates `input` at (blockX + warpX, blockY, blockZ + warpZ). |

These are the only custom **density function** nodes; everything else in the JSON is vanilla (noise, add, mul, spline, cache_2d, flat_cache, blend_density, range_choice, min, max, clamp, etc.).

### 2.3 Registered material condition

| JSON type              | Class                         | Purpose |
|------------------------|-------------------------------|---------|
| `larion:somewhat_steep`| SomewhatSteepMaterialCondition| Surface rule condition: “slope to a neighbor is &gt; 1 block” (uses SomewhatSteepSlopePredicate). |

Used in overworld noise_settings surface rules (e.g. for grass/stone placement on steep slopes).

## 3. Mixins and access

### 3.1 Mixin config

- **File**: `common/src/main/resources/larion.mixins.json`.
- **Package**: `com.badgerson.larion.mixin`.
- **Refmap**: `mixin.refmap.json` (generated by the loader’s mixin plugin).
- **Mixins** (all apply on both client and server where relevant):

| Mixin                     | Target                               | Purpose |
|---------------------------|--------------------------------------|---------|
| NoiseChunkGeneratorMixin  | `NoiseBasedChunkGenerator`           | Replaces `createFluidPicker` so lava sea level is `noiseSettings.minY() + 10` instead of vanilla -54; keeps water sea level from settings. |
| CaveWorldCarverMixin      | `CaveWorldCarver`                    | Overrides `getCaveBound()` to return `Constants.CAVE_BOUND_OVERRIDE` (40) instead of vanilla 15 (deeper cave carving). |
| SteepSlopePredicateMixin  | `SurfaceRules.Context.SteepMaterialCondition` | Replaces “steep” slope logic with Larion’s: WORLD_SURFACE_WG height at (x,z) vs 4 neighbors; steep if difference &gt; 3. |
| MaterialRuleContextMixin | `SurfaceRules.Context`              | In constructor TAIL: sets field `somewhatSteepSlopePredicate` = new SomewhatSteepSlopePredicate(context). Exposed via MaterialRuleContextExtensions. |
| SurfaceRulesContextAccessor | `SurfaceRules.Context` (interface) | Accessors: `blockX`, `blockZ`, `chunk` (so slope predicates can read block pos and chunk for heightmap). |

### 3.2 Access transformer (Forge)

- **common**: `accesstransformer-common.cfg` makes visible:
  - `DensityFunctions.NOISE_VALUE_CODEC`, `DensityFunctions.PureTransformer`
  - `SurfaceRules.LazyXZCondition`, `SurfaceRules.Condition`, `SurfaceRules.Context`, `SurfaceRules.Context.SteepMaterialCondition`
  - `SurfaceRules.Context` fields: `blockX`, `blockZ`, `chunk`

So Larion can extend `LazyXZCondition`, implement conditions, and read context position/chunk without reflection.

## 4. Worldgen data flow (high level)

### 4.1 Dimension and noise settings

- **Dimension types** (in `data/minecraft/dimension_type/`):
  - `overworld.json` / `overworld_caves.json`: height 640, min_y -128 (tall world).
- **Noise settings** (`data/minecraft/worldgen/noise_settings/overworld.json`):
  - `noise`: height 640, min_y -128, size_horizontal/vertical 1.
  - **noise_router**: maps logical names to density functions (or inline definitions). Key entries:
    - **continents** → `minecraft:overworld/continents`
    - **depth** → `minecraft:overworld/depth`
    - **erosion** → `minecraft:overworld/erosion`
    - **ridges** → `minecraft:overworld/ridges`
    - **final_density** → `larion:overworld/final_density`
    - **temperature** → `larion:overworld/temperature`
    - **vegetation** → `larion:overworld/vegetation`
    - Plus aquifer-related and other vanilla entries; some overwritten with `larion:overworld/...` (e.g. fluid_level_floodedness, fluid_level_spread).

Minecraft’s **NoiseBasedChunkGenerator** uses these noise settings. When it needs to fill terrain, it uses a **NoiseChunk** (or equivalent) that gets density from the **noise_router**. The **final_density** entry is the root used to decide solid vs air for each block in the chunk.

### 4.2 Redirect chain (minecraft → larion)

Vanilla overworld JSONs **redirect** to Larion’s definitions so that Larion controls the shapes while keeping names compatible:

- `minecraft:overworld/continents` → `flat_cache(cache_2d(larion:overworld/continents))`
- `minecraft:overworld/depth` → uses `larion:overworld/terrain/offset` etc.
- `minecraft:overworld/erosion` → `larion:overworld/erosion`
- `minecraft:overworld/ridges` → `flat_cache(cache_2d(larion:overworld/ridges))`
- `minecraft:overworld/sloped_cheese` → `cache_once(larion:overworld/sloped_cheese)`

So when the engine asks for “continents”, “depth”, “erosion”, “ridges”, or “sloped_cheese”, it ends up evaluating Larion’s density functions.

## 5. Density function tree (conceptual)

### 5.1 Final density

`larion:overworld/final_density` is the **root** density for terrain. Simplified structure:

- **squeeze(0.64 × interpolated(blend_density( … )))**
  - Inner add chain builds from:
    - **y_clamped_gradient** (base fill -128 to -104, 500 to 512).
    - **depth** (minecraft:overworld/depth → Larion depth/terrain).
    - **min(sloped_cheese, range_choice(sloped_cheese, … caves …), …)**:
      - **sloped_cheese** = Larion’s warped terrain (flat_domain_warp + depth + caves).
      - **range_choice** and **min** mix in **caves** (chasms, lava_river_noodle, cave_cheese, cave_layer, pillars, etc.).

So: **final_density** = vertical gradient + depth, blended and then combined with sloped_cheese and cave density; result is squeezed and interpolated. Positive density → solid, negative → air.

### 5.2 Continents

- **minecraft:overworld/continents** = `flat_cache(cache_2d(larion:overworld/continents))`.
- **larion:overworld/continents** = **spline**:
  - **coordinate**: `larion:flat_domain_warp` with:
    - **input**: `larion:overworld/continents/shapes` (noise-based “shapes” expression).
    - **warp_x** / **warp_z**: scale 800 × larion noise (continents/warp_x, continents/warp_z) at xz_scale 0.28.
  - **points**: spline (location → value) with one nested spline (continents/shroom coordinate).

So continents are a **spline of a warped 2D shape**: the “coordinate” is sampled at (x + warp_x, z + warp_z), then the spline maps that to a value. This produces continental blobs and warped coastlines.

### 5.3 Depth and terrain

- **depth** uses **larion:overworld/terrain/offset** (terrain base_height + features, with y_clamped_gradient).
- **Terrain** (offset, base_height, features, jaggedness, cliffs, hills, etc.) uses many **larion:** noises (terrain/jagged, terrain/cliffs_*, terrain/hills*, etc.) and some **larion:flat_domain_warp** in sloped_cheese.

### 5.4 Vegetation

- **vegetation** = `flat_cache(cache_2d(larion:overworld/vegetation))`.
- **larion:overworld/vegetation** = **flat_domain_warp**:
  - **input**: combination of vegetation noises (scale_a, scale_b, major) with add/mul/max/abs.
  - **warp_x** / **warp_z**: 600/400 × larion vegetation warp noises at xz_scale 0.12.

Used for biome-like variation (temperature/vegetation in the noise router).

### 5.5 Ridges

- **ridges** = `flat_cache(cache_2d(larion:overworld/ridges))`.
- **larion:overworld/ridges** = signum(ridges/weird) × min(spline(river_tunnel → rivers), max(0, spline(peaks))).
  - **weird**, **river_tunnel**, **rivers**, **peaks** themselves use **larion:flat_domain_warp** and various larion noises (ridges/weird, river_shift_*, rivers_*, peaks_*, etc.).

So ridges are warped river/peak splines with a signum mask.

### 5.6 Sloped cheese

- **minecraft:overworld/sloped_cheese** = `cache_once(larion:overworld/sloped_cheese)`.
- **larion:overworld/sloped_cheese** = 8 × quarter_negative(**flat_domain_warp**(min(river_noodle, depth), terrain warp_x/warp_z)).

So the “cheese” terrain is warped in X/Z by terrain noise and combined with depth and river noodle.

## 6. Execution path (when a chunk is generated)

### 6.1 Who calls what

1. **World generation** requests a chunk → **ChunkGenerator** (vanilla **NoiseBasedChunkGenerator**).
2. Generator uses **NoiseGeneratorSettings** (loaded from `overworld.json`), which includes the **noise_router**.
3. For each chunk (or cell), the engine builds a **NoiseChunk** (or equivalent) that implements **ContextProvider** and provides **DensityFunction.FunctionContext** per sample index.
4. The router’s **final_density** density function is evaluated over a 3D grid (typically vertical columns or full 3D) via:
   - **fillArray(double[] densities, ContextProvider provider)** for batch, or
   - **compute(FunctionContext context)** for single sample.
5. **final_density** is a tree of vanilla and Larion nodes. Every node that appears in the tree is evaluated (compute or fillArray) according to the engine’s traversal. So when the root is evaluated:
   - **sloped_cheese** is evaluated (cache_once), which triggers **larion:overworld/sloped_cheese** (flat_domain_warp, depth, river_noodle, etc.).
   - **continents**, **depth**, **erosion**, **ridges** are evaluated where referenced (e.g. in depth, initial_density_without_jaggedness, splines, etc.).
   - Each **larion:flat_domain_warp** does:
     - Fill warp arrays: `warpX.fillArray`, `warpZ.fillArray`.
     - Build a **WarpedContextProvider** (wraps engine context, overrides blockX/blockZ with warped values).
     - Call **input.fillArray(densities, warpedProvider)** so the whole subtree runs in batch.
   - **Division**, **Signum**, **Sine**, **Sqrt**, **XCoord**, **ZCoord** are evaluated wherever the JSON references them (e.g. ridges use signum; temperature uses larion:sine and larion:z).
6. Resulting density array is used to decide **block state** (solid vs air, then stone vs water/lava by aquifer). Then **biomes**, **surface rules**, and **carvers** run.

### 6.2 When mixins run

- **NoiseChunkGeneratorMixin**: When the chunk generator **creates the fluid picker** (aquifer) for the world; runs once per world/dimension setup. Replaces the picker so lava level follows minY + 10.
- **CaveWorldCarverMixin**: Whenever **CaveWorldCarver.getCaveBound()** is called during carving; returns 40 so caves can extend deeper.
- **SteepSlopePredicateMixin**: When surface rules evaluate the **vanilla “steep”** condition; replaces it with Larion’s heightmap-based steep check (threshold 3).
- **MaterialRuleContextMixin**: When a **SurfaceRules.Context** is constructed; adds the “somewhat steep” predicate to that context.
- **SurfaceRulesContextAccessor**: No “when”; it only exposes context fields when code casts context to the accessor interface (e.g. in SteepSlopePredicateMixin and SomewhatSteepSlopePredicate).

### 6.3 Data structures in the hot path

- **DensityFunction** tree: built once when the world loads (from JSON); then only **compute** / **fillArray** are called.
- **ContextProvider**: Implemented by the engine (e.g. NoiseChunk); provides **FunctionContext** per index. Larion’s **WarpedContextProvider** wraps it and overrides blockX/blockZ for flat_domain_warp.
- **Temp arrays**: In **Division.fillArray**: two arrays of length N (dividend, divisor). In **FlatDomainWarp.fillArray**: two arrays (warpX, warpZ). All per fillArray call; N = number of samples in the current batch (e.g. vertical column or chunk slice).
- **Caches**: Vanilla **flat_cache** and **cache_2d** cache results by (x,z) or by full context to avoid recomputation; Larion does not add new caches.

## 7. Summary table

| Layer            | What Larion adds / changes |
|------------------|----------------------------|
| **Mod entry**    | Registers 7 density function types + 1 material condition. |
| **Mixins**       | Fluid picker (lava level), cave bound (40), steep/somewhat steep surface logic, context accessors. |
| **Access**       | SurfaceRules.Context and LazyXZCondition/PureTransformer visible to common. |
| **Worldgen data**| Overworld dimension (640 height, -128 min_y), overworld noise_settings with noise_router pointing to larion:overworld/... and minecraft:overworld/... redirects to Larion density functions, custom noises under `larion:...`, surface rules using larion:somewhat_steep and larion:surface_cliff. |
| **Density logic**| final_density, continents, depth, erosion, ridges, sloped_cheese, vegetation, temperature, caves, etc., implemented as a big tree of vanilla + larion nodes; **flat_domain_warp** is the main Larion-specific node (domain warp in X/Z for fantasy shapes). |
